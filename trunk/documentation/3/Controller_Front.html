Using prefilters for Access Control

The pre-filters can do a number of things, such as DI. But in the Access Control use case they really are a rule based forward. If access is allowed then nothing is returned; if access is denied then a forward (array/string) is returned and the requested Action is not dispatched. The Front Controller loops and runs the forward. You can either have the Action Controller method return the forward or have the pre-filter return the forward. The easy way to have the Action Controller method return the forward is to pass a A_DL as the pre-filter so it just proxies the call to the Controller method. The other was it to have the pre-filter be a class of its own, do the check to see if the method exists, call the method if it does, then determine if a forward is necessary. And it does not have to be a method; it could be a property if you wanted to (but it would be public and not read only).

The key to these Access Control pre-filters is that they check if the method exists. So you only add code to the Controllers that you want to have Access Control. Compare that to a DI style pre-filter which usually checks for a specific class type and does the injection based on it knowledge of that class.
