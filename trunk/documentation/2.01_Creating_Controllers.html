Creating Controllers

Action Controllers are classes that are loaded, instantiated and run by the Front controller. They typically are located in a controllers/ directory. Each file must contain a class the same name. So the file controllers/foo.php should contain a class named foo.

class foobar extends A_Controller_Action {
	public function __construct($locator) {
		parent::__construct($locator);
	}
	public function run($locator) {
	}
	public function foo($locator) {
	}
	public function _bar() {
	}
}

The constructor and the dispatched method will be passed a Locator object by Front Controller. This parameter is optional and you may choose to use or or not.

The Constructor

The constructor is called when the controller class is instantiated in the Front Controller. This method is passed passed a Locator object by Front Controller. IMPORTANT - if your controller inherits a base class you need to call parent::__construct($locator).

The Default Method - run()

If the value of the 'action' Request parameter is null or does not match the name of a method then the default method will be dispatched. That means that mysite.com/foobar/ or mysite.com/foobar/xqz will call the run() method. This method is passed passed a Locator object by Front Controller.

Action Methods

If the value of the 'action' Request parameter matches the name of a method then it will be dispatched. That means that mysite.com/foobar/_bar/ will call the foo() method. This method is passed passed a Locator object by Front Controller.

Un-dispatchable Methods

Method name that start with an underscore cannot be dispatched by the Front Controller. In the example above the _bar() method cannot be dispatched. That means that mysite.com/foobar/_bar/ will call the run() method, not the _bar() method.

There are five ways to implement Action Controllers:

  1. no base class -- Skeleton's Front Controller will dispatch plain PHP classes
  2. inherit your own base class
  3. inherit A_Controller_Action for basic functionality
  4. inherit A_Controller_Action_Dispatch for pre/post-filter functionality, etc.
  5. inherit your own base class extend A_Controller_Action or A_Controller_Action_Dispatch

Types of Controllers

No Base Class

Skeleton's Front Controller will dispatch plain PHP classes. So you can create an Action Controller like this: 

class foobar {
	public function run() {
		// called for no action mysite.com/foobar/ or missing action mysite.com/foobar/xxx
	}
	public function foo() {
		// called for mysite.com/foobar/foo/
	}
	public function bar() {
		// called for mysite.com/foobar/bar/
	}
}

Inherit Your Own Base Class

Because Skeleton can dispatch any properly named class, you can create your own base class for groups of controllers.

Inherit A_Controller_Action for basic functionality

The basic Skeleton Action Controller provides a forward() and load() methods. The forward() method is for telling the Front Controller to run another controller after returning from the dispatched method. The load() method loads instances of models, views, templates, helpers, etc. objects.

Inherit A_Controller_Action_Dispatch

The Dispatch Action Controller adds for pre/post-filter functionality to the base Action Controller. You can implement methods in you Action Controller class that are called 1) after __construct() but before the dispatched method is called, or 2) after the dispatched method exits but before returning control to the Front Controller.

This controller implement a dispatch() method that is called directly by the Front Controller. The dispatch() method in turn calls any pre-filters first, then the dispatched method, and then any post-filters.

Extend A_Controller_Action or A_Controller_Action_Dispatch

You may extend the Skeleton Action Controllers to add additional functionality as a base class. You would do this if you wanted to provide this addtionality to more than on of your Action Controllers.
