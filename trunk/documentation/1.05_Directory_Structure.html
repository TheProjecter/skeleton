Directory Structure

There is no mandatory directory structure in Skeleton. All directory paths are customizable. However, the framework does have default directory names. There a also recommended directory structure.

You may set the directories for controllers and modules in the A_Controller_Mapper using the setBaseDir() for the path to the application, setDefaultDir() for the path to the default module, and setClassDir() for the path to the controllers. Typically the base dire will be an absolute and the module and controllers paths will be relative to it.

Minimal Structure

Assuming that you are using the Front Controller, in its most basic configuration only a 'controllers' directory is set as a default. 

controllers/

If you choose to base your action controller classes on one of the Skeleton Action Controllers and use the load() functionalty, then 'models', 'views', 'templates' and 'helpers' directories are defaults. So a directory structure for a simple application might be: 

controllers/
helpers/
models/
templates/
views/

Recommended Structures

You can specify an application directory to contain all of the MVC directories. The default application directoyr is './'. You specify this directory using the A_Controller_Mapper call with $Mapper->setBasePath('/path/to/app');

/app 
/app/controllers
/app/models
/app/views

You can further organize your application ito module directories by adding a 'module' parameter in the Request. Specify a default module directory the mapper with $Mapper->setDefaultDir('default');

/app
/app/default
/app/default/controller
/app/default/model
/app/default/view
/app/mymodule/controller
/app/mymodule/model
/app/mymodule/view
